# Contributing to CodePulse

Thanks for your interest in contributing! 🎉

## Development Setup

```bash
git clone https://github.com/ARYAN007H/codepulse.git
cd codepulse
python -m venv .venv
source .venv/bin/activate  # or .venv\Scripts\activate on Windows
pip install -e .
```

## Guidelines

- **Code style**: Follow PEP 8. Use type hints where possible.
- **Commits**: Write clear, descriptive commit messages.
- **Testing**: Test your changes on at least one real-world repo before submitting.
- **HTML template**: If modifying `template.py`, ensure the report works offline with no console errors.

## Reporting Bugs

Open an issue with:
1. Your Python version (`python --version`)
2. Your OS
3. Steps to reproduce
4. Expected vs actual behavior

## Feature Requests

Open an issue tagged `[Feature Request]` describing:
- The problem you're trying to solve
- Your proposed solution
- Any alternatives you've considered

## Pull Requests

1. Fork the repo
2. Create a feature branch (`git checkout -b feature/amazing-thing`)
3. Make your changes
4. Test on a real repo
5. Submit a PR with a clear description

Thank you! 🚀
